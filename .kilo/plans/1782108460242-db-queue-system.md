# DB-Backed Queue System

## Summary
Replace the local Vue `queue` ref in `Dashboard.vue` with a persisted database-backed queue system using named queue sets, JSON snapshots of items, and full CRUD API.

## Data Model

### `queues` table
- `id` — auto-inc PK
- `name` — string (queue set name, e.g. "Morning Service")
- timestamps

### `queue_items` table
- `id` — auto-inc PK
- `queue_id` — FK → queues, cascade on delete
- `name` — string (display label for the item)
- `queue` — json (full self-contained snapshot)
- `position` — integer, default 0
- timestamps

### JSON snapshot format
**Lower third:**
```json
{ "type": "lowerthird", "item_id": 5, "name": "John Doe", "subtitle": "Speaker", "template": "classic", "image": null }
```
**Song (full):**
```json
{ "type": "lyrics", "song_id": 3, "title": "Amazing Grace", "artist": "John Newton", "slides": [{"slide_order":0,"content":"...","section_label":"Verse 1"},...] }
```

## New Files

| File | Notes |
|---|---|
| `database/migrations/..._create_queues_table.php` | `id, name, timestamps` |
| `database/migrations/..._create_queue_items_table.php` | `id, queue_id(FK cascade), name, queue(json), position(unsignedInt default 0), timestamps` |
| `app/Models/Queue.php` | `hasMany QueueItem`, `$fillable: [name]` |
| `app/Models/QueueItem.php` | `belongsTo Queue`, `$fillable: [name, queue, position]`, casts `queue` as array |
| `app/Http/Controllers/Api/QueueController.php` | Full CRUD + items sub-resource + move endpoint |
| `app/Http/Resources/QueueResource.php` | Returns queue with nested items, items include the snapped-out `queue` JSON |

## API Endpoints

All under `Route::middleware(['web','auth'])` → prefix no prefix, just resourceful routes on `/api/queues`.

| Method | URI | Action |
|---|---|---|
| GET | `/api/queues` | `index` — list all queues |
| POST | `/api/queues` | `store` — create queue `{name}` |
| GET | `/api/queues/{queue}` | `show` — queue with items ordered by position |
| PUT | `/api/queues/{queue}` | `update` — rename `{name}` |
| DELETE | `/api/queues/{queue}` | `destroy` — cascade deletes items |
| POST | `/api/queues/{queue}/items` | `addItem` — `{name, queue}` auto-assigns position |
| PUT | `/api/queues/{queue}/items/{item}` | `updateItem` — edit name/queue data |
| DELETE | `/api/queues/{queue}/items/{item}` | `removeItem` — reindex positions after |
| PATCH | `/api/queues/{queue}/items/{item}/move` | `moveItem` — `{direction: "up"|"down"}`, swaps positions |

### Validation rules
- `addItem`: `name` required|string, `queue` required|array, `queue.type` required|in:lowerthird,lyrics
- `moveItem`: `direction` required|in:up,down

## Frontend Changes (`resources/js/Pages/Control/Dashboard.vue`)

### State (replaces old `queue` ref)
```ts
const queues = ref<QueueSet[]>([])            // list of all queue sets
const selectedQueueId = ref<number | null>(null)
const queueItems = ref<QueueItemResource[]>([])  // items of selected queue
```

### New functions to add
- `fetchQueues()` — load queue list on mount
- `selectQueue(id)` — load items for a queue, set selectedQueueId
- `createQueue()` — prompt for name, POST, refresh list
- `deleteQueue(id)` — confirm, DELETE, refresh
- `renameQueue()` — prompt, PUT
- `addToQueueApi(type, data)` — builds JSON snapshot, POSTs to selected queue; auto-creates "Untitled Queue" if none selected
- `removeFromQueueApi(itemId)` — DELETE
- `moveQueueItem(itemId, direction)` — PATCH, re-fetch items

### Queue tab template
- Dropdown to select/create queues: `<select>` + buttons [+ New] [Rename] [Delete]
- If no queue selected: message "Select or create a queue"
- If queue selected but empty: "Queue is empty — add items from Lower Thirds or Lyrics tabs"
- Item list: each row shows type badge, name, subtitle/artist, with [Show] [▲] [▼] [Edit] [Remove] buttons
- Edit: inline input or prompt to change item display name

### "Add to Queue" buttons
- **Lower third cards** — existing Queue button → call `addToQueueApi('lowerthird', lt)`
- **Song list cards** — existing Queue button → call `addToQueueApi('lyrics', song)`
- **Song detail view (slides preview)** — NEW Queue button in the header bar (next to "Send to Display") → call `addToQueueApi('lyrics', selectedSong)`

### Edge cases handled
- No queue selected on "Add to Queue" click → auto-create "Untitled Queue" via API, then add item
- Original lower third/song deleted from DB → snapshot still displays in queue; Show button checks if original exists in local list and disables if missing
- Move up on first item or move down on last item → no-op (API returns 422 or ignores)
